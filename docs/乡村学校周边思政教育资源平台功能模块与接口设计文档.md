# 乡村学校周边思政教育资源平台功能模块与接口设计文档

## 1. 文档说明

本文档在 [乡村学校周边思政教育资源数字地图平台需求文档.md](/D:/大创项目/乡村学校周边思政教育资源数字地图平台需求文档.md) 基础上，进一步细化平台的：

1. 功能模块设计
2. 数据库设计要点
3. 接口设计建议
4. MVP 开发优先级

本文档尽量与当前项目已有代码结构保持一致，便于直接指导后续开发和答辩展示。

## 2. 设计目标

本阶段设计的核心目标不是重做原系统，而是在现有“红色文化地图平台”基础上，补齐“学校中心化”的业务闭环。

设计原则：

1. 保留现有红色文化数据库与地图能力
2. 新增学校主轴与通用教育资源主轴
3. 先做试点学校最小闭环，再逐步扩展
4. 功能设计优先服务地图展示、资源组织和教学落地

## 3. 平台总体模块划分

结合项目目标，平台建议划分为 6 个一级模块。

### 3.1 学校管理模块

定位：

负责学校基础信息、坐标信息和学校主数据管理。

核心任务：

1. 录入与维护学校基础信息
2. 采集并校验学校坐标
3. 支持按地区、学段、关键词检索学校
4. 为后续地图展示和资源关联提供基础对象

### 3.2 本土思政教育资源管理模块

定位：

负责统一管理学校周边可用于思政教育的各类资源。

核心任务：

1. 录入资源基础信息
2. 建立统一资源分类体系
3. 维护资源的教育价值、适用学段、活动建议等字段
4. 将红色文化资源与非遗、公益、传统文化等内容统一纳入资源池

### 3.3 学校周边关联模块

定位：

负责建立“学校与资源之间”的结构化关系。

核心任务：

1. 标识哪些资源属于某所学校周边可用资源
2. 记录距离、可达性、出行方式、活动时长等信息
3. 形成围绕学校的资源清单

### 3.4 地图展示与检索模块

定位：

通过地图交互展示学校、周边资源及其关系。

核心任务：

1. 展示学校分布
2. 展示资源分布
3. 支持学校详情查看
4. 支持周边资源检索与地图联动

### 3.5 教学主题与活动建议模块

定位：

负责把资源转化为教学主题与活动方案。

核心任务：

1. 对资源进行主题化组织
2. 围绕学校输出教学活动建议
3. 支持不同年级和不同类型活动组织

### 3.6 数据采集与审核模块

定位：

负责平台数据治理与质量控制。

核心任务：

1. 支持采集导入
2. 支持来源记录
3. 支持审核状态管理
4. 支持数据追踪与修正

## 5. 数据库设计说明

## 5.1 总体思路

数据库设计建议采用“增量扩展”方式，而不是推翻现有红色文化库。

设计结构建议为：

1. 保留原红色文化核心表
2. 新增学校主轴表
3. 新增通用教育资源表
4. 新增学校与资源关系表
5. 新增教学活动建议表
6. 复用原有标签、媒体、来源、内容分块、审计日志等通用支撑表

## 5.2 核心新增表

结合现有 SQL 文件，建议以以下 5 张表为第二阶段核心：

1. `school`
2. `school_geo_record`
3. `local_edu_resource`
4. `school_resource_rel`
5. `teaching_activity_plan`

## 5.3 核心表关系

### 5.3.1 school

平台学校主表。

作用：

1. 存学校基础信息
2. 存学校坐标
3. 作为地图展示和资源关联入口

### 5.3.2 school_geo_record

学校坐标历史与审核表。

作用：

1. 记录坐标来源
2. 记录人工审核结果
3. 支持后续修订追踪

### 5.3.3 local_edu_resource

通用教育资源主表。

作用：

1. 统一管理不同类型教育资源
2. 支持红色文化与非红色资源并存
3. 为地图和教学活动提供资源基础

### 5.3.4 school_resource_rel

学校与资源关系表。

作用：

1. 表示某学校周边可用资源
2. 存储距离、交通、优先级、主题摘要等信息

### 5.3.5 teaching_activity_plan

教学活动方案表。

作用：

1. 将资源组织为教学方案
2. 支持按学校输出活动建议

## 5.4 与现有旧表的复用关系
