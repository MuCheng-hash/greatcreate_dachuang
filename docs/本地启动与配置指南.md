# 红韵乡途本地启动与配置指南

本文档面向 Windows 本地开发。项目当前由 Spring Boot 业务服务和一套 FastAPI Stateful Agent 服务组成；MySQL 是必需依赖，Qdrant、Neo4j 和真实模型均可按需启用。

## 1. 服务关系与启动顺序

| 服务 | 默认地址 | 是否必需 | 负责内容 |
| --- | --- | --- | --- |
| MySQL 8.0 | localhost:3306 | 是 | 学校、资源、认证、教学方案等主业务数据 |
| Spring Boot | http://localhost:8080 | 是 | 教师门户、管理员后台、认证、地图、RAG 入口 |
| FastAPI Agent | http://127.0.0.1:5050 | 问答/AI 功能需要 | Stateful Agent、教学方案、模型调用与降级 |
| Qdrant | http://127.0.0.1:6333 | 真实 RAG 需要 | 内容分块向量索引与 ANN 检索 |
| Neo4j | bolt://127.0.0.1:7687 | 图谱功能需要 | 图谱查询与关系事实 |
| Ollama | http://127.0.0.1:11434 | 可选 | 本地降级/轻量模型 |

推荐启动顺序：

1. MySQL；
2. Spring Boot；
3. FastAPI Agent；
4. 需要真实 RAG 时再启动 Qdrant 并重建索引；
5. 需要图谱事实时再启动 Neo4j 并同步数据。

最小可运行组合只有 MySQL 和 Spring Boot。此时可使用地图、注册、审核、登录和已有正式资源；AI 功能需要 FastAPI，未配置模型时 FastAPI 会返回标记为 degraded 的本地结构化兜底结果。

## 2. 前置环境

在新的 PowerShell 窗口中先设置 UTF-8 输出，再检查工具版本：

~~~powershell
$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::new($false)
java -version
mvn -version
mysql --version
python --version
~~~

| 依赖 | 要求 | 用途 |
| --- | --- | --- |
| JDK | 21 | 业务服务编译和启动 |
| Maven | 可执行 mvn | 项目没有 Maven Wrapper |
| MySQL | 8.0 | 必需主数据库 |
| Python | 3.9 或更高 | FastAPI 服务和 Neo4j 同步脚本 |
| Docker Desktop | 启动 Qdrant 时需要 | 本机未安装 Docker 时不能执行 compose 命令 |
| Neo4j | 可选 | 图谱查询 |
| Ollama | 可选 | 本地模型降级 |

如果系统默认 Java 不是 JDK 21，在启动 Maven 前为当前窗口设置正确路径：

~~~powershell
$env:JAVA_HOME = "D:/java/jdk-21"
$env:Path = "$env:JAVA_HOME/bin;$env:Path"
java -version
~~~

将上例路径替换为本机实际 JDK 21 目录。

## 3. 初始化 MySQL

全新、本地且允许清空数据的数据库，可以导入全量脚本。该脚本会删除并重建项目相关表，**不能**用于已有有效数据的数据库。

~~~powershell
mysql --default-character-set=utf8mb4 -u root -p
~~~

进入 MySQL 客户端后，使用仓库绝对路径导入：

~~~sql
SOURCE D:/Code/java/javaCode/greatcreate_dachuang/data/sql/mysql_red_culture_all_in_one.sql;
~~~

全量脚本会创建 red_culture_platform 数据库、表结构、认证表和样例数据。导入成功后，不要再对同一个新库重复执行 data/sql 下的拆分建表脚本。

已有开发库需要升级周边 AI 资源发现功能时，执行增量脚本而不是全量重置：

~~~sql
SOURCE D:/Code/java/javaCode/greatcreate_dachuang/data/sql/mysql_ai_poi_resource_discovery.sql;
~~~

## 4. 配置并启动业务服务

业务服务配置文件是 business-service/src/main/resources/application.yml。配置可以只作用于当前 PowerShell 窗口，也可以保存为当前 Windows 用户的长期环境变量。不要把个人密码和密钥写入仓库。

### 4.1 必需配置

MySQL 默认值是 localhost:3306、red_culture_platform、root、root。本机不同则设置 Spring 标准环境变量：

~~~powershell
$env:SPRING_DATASOURCE_URL = "jdbc:mysql://127.0.0.1:3306/red_culture_platform?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
$env:SPRING_DATASOURCE_USERNAME = "root"
$env:SPRING_DATASOURCE_PASSWORD = "替换为你的MySQL密码"
~~~

本地演示管理员可自定义；刚导入全量脚本且未覆盖时默认账号为 admin，默认密码为 admin123456。

~~~powershell
$env:APP_ADMIN_USERNAME = "admin"
$env:APP_ADMIN_PASSWORD = "替换为本地开发密码"
$env:APP_ADMIN_DISPLAY_NAME = "平台管理员"
~~~

JWT 密钥和 Agent 内部令牌在本地也建议显式设置。Agent 内部令牌必须在启动 Spring Boot 和 FastAPI 的两个 PowerShell 窗口中设置为**同一个值**。

~~~powershell
$env:APP_AUTH_JWT_SECRET = "至少32字节的本地随机字符串"
$env:AGENT_INTERNAL_SERVICE_TOKEN = "本地Agent服务间共享令牌"
$env:LLM_SERVICE_BASE_URL = "http://127.0.0.1:5050"
~~~

高德地图配置是可选的。未配置 AMAP_WEB_SERVICE_KEY 时，已审核正式资源仍可展示，但不会执行服务端周边 POI 发现。

~~~powershell
$env:AMAP_WEB_KEY = "高德地图Web端Key"
$env:AMAP_SECURITY_JS_CODE = "高德安全密钥"
$env:AMAP_WEB_SERVICE_KEY = "高德Web服务Key"
~~~

### 4.2 长期保存环境变量（推荐）

$env:变量名 = "值" 只对当前 PowerShell 窗口及其启动的子进程有效。关闭窗口后，下一次打开的终端、IDE 或服务进程不会自动获得这些变量。

如果希望配置长期保留，使用 Windows 当前用户级环境变量。下面的命令不需要管理员权限，保存后对以后启动的程序生效：

~~~powershell
[Environment]::SetEnvironmentVariable("AMAP_WEB_SERVICE_KEY", "替换为高德Web服务Key", "User")
~~~

注意：

- 示例中的“替换为……”只是占位文字；如果你习惯写成 <高德Web服务Key>，实际执行时不要把 <> 一起写入。
- 修改用户级环境变量后，必须关闭并重新打开 PowerShell、IDE 的终端窗口，然后再重启 Spring Boot 和 FastAPI。已经运行的进程不会自动读取新值。
- User 表示当前 Windows 账号；不要随意改成 Machine，后者需要管理员权限并会影响整台电脑。

可以一次性保存本项目常用配置。请先把示例中的占位文字替换成自己的值，再执行：

~~~powershell
$userEnv = @{
  "SPRING_DATASOURCE_URL" = "jdbc:mysql://127.0.0.1:3306/red_culture_platform?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
  "SPRING_DATASOURCE_USERNAME" = "root"
  "SPRING_DATASOURCE_PASSWORD" = "替换为你的MySQL密码"
  "APP_ADMIN_USERNAME" = "admin"
  "APP_ADMIN_PASSWORD" = "替换为本地开发密码"
  "APP_ADMIN_DISPLAY_NAME" = "平台管理员"
  "APP_AUTH_JWT_SECRET" = "替换为至少32字节的随机字符串"
  "AGENT_INTERNAL_SERVICE_TOKEN" = "替换为一段随机长字符串"
  "LLM_SERVICE_BASE_URL" = "http://127.0.0.1:5050"
  "INTERNAL_BUSINESS_BASE_URL" = "http://127.0.0.1:8080"
  "AMAP_WEB_KEY" = "替换为高德地图Web端Key"
  "AMAP_SECURITY_JS_CODE" = "替换为高德安全密钥"
  "AMAP_WEB_SERVICE_KEY" = "替换为高德Web服务Key"
  "RAG_ENABLED" = "true"
  "RAG_EMBEDDING_BASE_URL" = "https://dashscope.aliyuncs.com/compatible-mode/v1"
  "RAG_EMBEDDING_API_KEY" = "替换为百炼Embedding Key"
  "RAG_EMBEDDING_MODEL" = "text-embedding-v3"
  "RAG_EMBEDDING_DIMENSIONS" = "1024"
  "QDRANT_BASE_URL" = "http://127.0.0.1:6333"
  "QDRANT_COLLECTION" = "red_culture_content_chunks"
  "NEO4J_URI" = "bolt://127.0.0.1:7687"
  "NEO4J_USERNAME" = "neo4j"
  "NEO4J_PASSWORD" = "替换为Neo4j密码"
}

foreach ($item in $userEnv.GetEnumerator()) {
  [Environment]::SetEnvironmentVariable($item.Key, $item.Value, "User")
}
~~~

如果暂时只需要最小启动，可以只保存 MySQL、JWT、Agent 令牌和服务地址；高德、RAG、Qdrant、Neo4j 均可后续再补。重复执行这段命令会覆盖同名的当前用户环境变量。

真实模型链路是可选项。如果你已经有本地 Ollama 模型，不要执行 ollama pull；先用 ollama list 查看本机已有模型名，再只保存对应配置：

~~~powershell
[Environment]::SetEnvironmentVariable("AGENT_FALLBACK_PROVIDER", "ollama", "User")
[Environment]::SetEnvironmentVariable("AGENT_FALLBACK_MODEL", "替换为ollama list中的已有模型名", "User")
[Environment]::SetEnvironmentVariable("AGENT_FALLBACK_BASE_URL", "http://127.0.0.1:11434/v1", "User")
[Environment]::SetEnvironmentVariable("AGENT_FALLBACK_API_KEY", "ollama", "User")
~~~

如果要启用百炼主模型，再额外配置以下变量；不配置时可以继续使用本地模型或本地结构化降级：

~~~powershell
[Environment]::SetEnvironmentVariable("AGENT_PRIMARY_PROVIDER", "bailian", "User")
[Environment]::SetEnvironmentVariable("AGENT_PRIMARY_MODEL", "qwen-plus", "User")
[Environment]::SetEnvironmentVariable("AGENT_PRIMARY_BASE_URL", "https://dashscope.aliyuncs.com/compatible-mode/v1", "User")
[Environment]::SetEnvironmentVariable("AGENT_PRIMARY_API_KEY", "替换为百炼API Key", "User")
~~~

不要直接打印密码、令牌或 API Key 来验证配置。可以只检查变量是否存在及长度：

~~~powershell
$checkNames = @(
  "SPRING_DATASOURCE_PASSWORD",
  "APP_AUTH_JWT_SECRET",
  "AGENT_INTERNAL_SERVICE_TOKEN",
  "AMAP_WEB_SERVICE_KEY",
  "RAG_EMBEDDING_API_KEY"
)

foreach ($name in $checkNames) {
  $value = [Environment]::GetEnvironmentVariable($name, "User")
  [PSCustomObject]@{
    Name = $name
    Configured = -not [string]::IsNullOrWhiteSpace($value)
    Length = if ($null -eq $value) { 0 } else { $value.Length }
  }
}
~~~

删除某一个当前用户环境变量：

~~~powershell
[Environment]::SetEnvironmentVariable("变量名", $null, "User")
~~~

用户级环境变量会被当前 Windows 账号下启动的程序读取，因此不要在共享电脑上保存生产密钥；生产环境应使用专门的密钥管理服务。任何包含真实密码、令牌或 API Key 的 .env 文件都不要提交到 Git。

### 4.3 启动命令

在仓库根目录打开一个 PowerShell 窗口：

~~~powershell
$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::new($false)
Set-Location "D:/Code/java/javaCode/greatcreate_dachuang/business-service"
mvn spring-boot:run
~~~

Maven 会安装固定 Node 版本、执行前端依赖安装并构建教师门户。如果本机网络导致前端依赖安装失败，而仓库中已有静态门户产物，可临时跳过该步骤：

~~~powershell
mvn spring-boot:run "-Dskip.npm=true"
~~~

业务服务就绪后访问：

- 教师门户：http://localhost:8080/login
- 地图首页：http://localhost:8080/
- 管理后台：http://localhost:8080/admin.html
- 当前登录状态：http://localhost:8080/api/auth/me

未登录访问 /api/auth/me 返回 401 属于正常现象；登录成功后应返回当前账号信息。

## 5. 启动 FastAPI Stateful Agent

只保留一套 FastAPI 服务。llm-service/app.py 是兼容启动入口，实际应用工厂是 llm_service.api:create_app；不要另行启动旧 Flask 或第二个 Agent 进程。

在第二个 PowerShell 窗口执行：

~~~powershell
$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::new($false)
Set-Location "D:/Code/java/javaCode/greatcreate_dachuang/llm-service"
python -m venv ".venv"
& ".venv/Scripts/Activate.ps1"
python -m pip install -r "requirements.txt"

$env:APP_ENV = "dev"
$env:INTERNAL_BUSINESS_BASE_URL = "http://127.0.0.1:8080"
$env:AGENT_INTERNAL_SERVICE_TOKEN = "与业务服务完全相同的共享令牌"
python app.py
~~~

如果已经按 4.2 保存了用户级环境变量，重新打开 PowerShell 后可以省略上述同名 $env: 临时配置；仍需确认 Spring Boot 和 FastAPI 使用的是同一个 AGENT_INTERNAL_SERVICE_TOKEN。

如果 PowerShell 策略禁止激活虚拟环境，可直接使用虚拟环境解释器：

~~~powershell
& ".venv/Scripts/python.exe" -m pip install -r "requirements.txt"
& ".venv/Scripts/python.exe" "app.py"
~~~

基础就绪检查：

~~~powershell
Invoke-RestMethod -Uri "http://127.0.0.1:5050/health/live"
Invoke-RestMethod -Uri "http://127.0.0.1:5050/health/ready"
~~~

不配置外部模型时，Agent 会持久化会话并返回 degraded 结果；这适合先验证 Java 到 FastAPI 的联通、SSE 和工具调用，不代表真实模型已生效。

## 6. 配置真实模型和 Ollama 降级

推荐模型链路为“百炼 qwen-plus → Ollama 本地已有模型 → 本地结构化兜底”。以下变量可以在 FastAPI 窗口中临时设置，也可以按 4.2 保存为用户级环境变量：

~~~powershell
$env:AGENT_PRIMARY_PROVIDER = "bailian"
$env:AGENT_PRIMARY_MODEL = "qwen-plus"
$env:AGENT_PRIMARY_BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
$env:AGENT_PRIMARY_API_KEY = "替换为百炼API Key"

$env:AGENT_FALLBACK_PROVIDER = "ollama"
$env:AGENT_FALLBACK_MODEL = "替换为ollama list中的已有模型名"
$env:AGENT_FALLBACK_BASE_URL = "http://127.0.0.1:11434/v1"
$env:AGENT_FALLBACK_API_KEY = "ollama"
~~~

使用 Ollama 前确认服务和已有模型：

~~~powershell
ollama list
~~~

本指南不会要求下载模型；只有在本机确实没有所需模型、并且你明确决定下载时，才自行执行 ollama pull 模型名。

如果只配置旧的 OpenAI-compatible 参数，服务仍兼容：

~~~powershell
$env:LLM_API_URL = "https://provider.example/v1/chat/completions"
$env:LLM_API_KEY = "替换为模型API Key"
$env:LLM_MODEL = "模型名称"
~~~

真实密钥只能放在当前会话、受保护的用户级环境变量或密钥管理服务中，不要写入 README、.env 或提交到 Git。

## 7. 启用真实 RAG

RAG 使用 content_chunk、Embedding 和 Qdrant。默认 RAG_ENABLED 为 false；只有完成下面步骤后，教学方案和 Agent 才会使用向量召回与混合重排。

### 7.1 启动 Qdrant

Docker Desktop 已安装并启动时，在仓库根目录执行：

~~~powershell
docker compose -f docker-compose.rag.yml up -d
Invoke-RestMethod -Uri "http://127.0.0.1:6333/healthz"
~~~

当前机器未检测到 docker 命令。安装并启动 Docker Desktop 后再执行上面的 compose 命令；未安装 Docker 时保持 RAG 关闭，系统会使用明确标记的 keyword-fallback。

### 7.2 配置 Embedding 并重启业务服务

在启动业务服务的 PowerShell 窗口设置，或按 4.2 保存为用户级环境变量：

~~~powershell
$env:RAG_ENABLED = "true"
$env:RAG_EMBEDDING_BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
$env:RAG_EMBEDDING_API_KEY = "替换为百炼Embedding API Key"
$env:RAG_EMBEDDING_MODEL = "text-embedding-v3"
$env:RAG_EMBEDDING_DIMENSIONS = "1024"
$env:QDRANT_BASE_URL = "http://127.0.0.1:6333"
$env:QDRANT_COLLECTION = "red_culture_content_chunks"
~~~

随后启动或重启 Spring Boot。默认 RAG_SYNC_ON_STARTUP 为 true，会为已有 content_chunk 建立或更新向量索引。

### 7.3 管理员手动重建索引

POST /api/admin/rag/reindex 受管理员认证和 CSRF 保护，不能直接发送裸 POST。以下示例在本地新库默认管理员账号的前提下，先登录、保存 Cookie，再携带 XSRF Token 重建索引：

~~~powershell
$loginBody = @{ username = "admin"; password = "admin123456" } | ConvertTo-Json
Invoke-WebRequest -Uri "http://localhost:8080/api/auth/login" -Method Post -ContentType "application/json; charset=utf-8" -Body $loginBody -SessionVariable session | Out-Null
$csrf = ($session.Cookies.GetCookies("http://localhost:8080") | Where-Object { $_.Name -eq "XSRF-TOKEN" }).Value
Invoke-RestMethod -Uri "http://localhost:8080/api/admin/rag/reindex" -Method Post -WebSession $session -Headers @{ "X-CSRF-TOKEN" = $csrf }
~~~

如已修改管理员账号或密码，请同步替换 loginBody。成功响应会返回索引统计；失败时先检查 Qdrant、Embedding Key、模型维度和业务服务日志。

## 8. 启用 Neo4j 图谱

Neo4j 不是最小启动依赖。需要图谱查询时，在启动 Spring Boot 前设置，或按 4.2 保存为用户级环境变量：

~~~powershell
$env:NEO4J_URI = "bolt://127.0.0.1:7687"
$env:NEO4J_USERNAME = "neo4j"
$env:NEO4J_PASSWORD = "替换为Neo4j密码"
~~~

MySQL 到 Neo4j 的同步脚本位于 scripts/sync_mysql_to_neo4j.py。该脚本当前是本地原型，配置写在顶部的 MYSQL_CONFIG 和 NEO4J_CONFIG 中；运行前请使其与本机一致，且不要把包含真实密码的改动提交到 Git。

~~~powershell
Set-Location "D:/Code/java/javaCode/greatcreate_dachuang"
python -m pip install pymysql neo4j
python "scripts/sync_mysql_to_neo4j.py"
~~~

同步完成后，Spring Boot 的 TownGraphQueryService 会查询 Neo4j；未同步或 Neo4j 不可用时，业务会返回降级状态而非伪造图谱事实。

## 9. 验收清单

按所启用的层级验证，不要仅以端口监听作为成功标准：

1. 最小业务：MySQL 可连接；Spring Boot 可打开教师门户和管理员后台；学校登录后可查看地图资源。
2. FastAPI：/health/ready 返回就绪；智能问答能获得 run.started、tool 事件、token、final、done 等 SSE 事件。
3. 真实模型：回答不再是 degraded，并能在日志或观测接口中看到模型、耗时和 token 信息。
4. RAG：RAG 重建成功；教学方案或问答响应的 retrievalMethod 为 vector+hybrid-rerank，而不是 keyword-fallback。
5. Neo4j：同步脚本完成；包含关联事实的地图或问答可返回图谱事实。

## 10. 常见问题

| 现象 | 优先检查 |
| --- | --- |
| Maven 编译报 Java 版本错误 | 当前 JAVA_HOME 是否指向 JDK 21 |
| Spring Boot 无法连接 MySQL | 数据库是否导入、SPRING_DATASOURCE_* 是否正确、账号是否有权限 |
| 页面登录后立刻失效 | 必须从 localhost:8080 同源访问，不能双击 HTML 或改用其他端口 |
| Agent 返回 degraded | FastAPI 是否运行；Java 与 FastAPI 的 AGENT_INTERNAL_SERVICE_TOKEN 是否相同；模型是否已配置 |
| FastAPI 401 | 内部调用令牌不一致，或直接从浏览器伪造调用了受保护 Agent 接口 |
| RAG 仍是 keyword-fallback | RAG_ENABLED、Qdrant、Embedding Key、集合维度和索引状态未全部满足 |
| Docker 命令不存在 | 安装并启动 Docker Desktop，或暂不启用 Qdrant/RAG |
| Neo4j 图谱为空 | 先确认 Bolt 服务，再执行 MySQL 到 Neo4j 同步脚本 |
| 新配置没有生效 | 用户级环境变量只对新启动的进程生效；关闭并重新打开终端/IDE，再重启对应服务 |

## 11. 本机当前状态

本节只记录文档编写时的开发机状态，可能随时间变化。启动前请以实际端口和健康检查结果为准。

这些状态是本机开发状态，不代表其他开发者或部署环境的状态。
