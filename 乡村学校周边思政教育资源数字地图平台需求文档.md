# 乡村学校周边思政教育资源数字地图平台需求文档

## 1. 文档说明

本文档面向项目第二阶段建设，基于项目由“红色革命文化资源整理与展示”向“河北地区乡村学校周边思政教育资源数字地图平台”转型的实际需求编制。

文档目标：

1. 明确平台建设定位与边界
2. 统一后续功能设计、数据采集、数据库扩展和页面开发方向
3. 为项目立项汇报、中期检查、答辩展示和团队协作提供统一依据

## 2. 项目背景

前期项目主要围绕红色革命文化资源展开，重点完成了红色遗址、革命人物、历史事件、纪念馆、红色故事等内容的结构化整理、地图展示与知识图谱建模。

随着项目推进，平台目标不再局限于红色文化资源展示，而是面向河北地区乡村基础教育实际需求，进一步建设一个以“乡村学校”为核心坐标、以“学校周边真实教育资源”为主要内容、以“思政教学落地”为导向的数字地图平台。

平台希望解决的问题包括：

1. 乡村学校思政教育与本地真实资源结合不够紧密
2. 教师难以系统发现学校周边可用于思政教育的文化资源、社会资源和实践场所
3. 已有资源多停留在静态展示层，尚未形成可支持课堂教学和实践活动组织的结构化能力

## 3. 项目定位

本项目定位为：

**以河北省乡村学校为空间中心，以学校周边思政教育资源为内容主体，以课堂教学、实践活动与育人支持为应用目标的数字地图平台。**

平台不是单纯做一张资源分布图，而是要形成“学校位置精准化、周边资源可视化、教育主题可组织化、课堂活动可落地化”的一体化平台能力。

## 4. 建设目标

### 4.1 总体目标

构建覆盖河北地区乡村学校的思政教育资源数字地图平台，建立学校、周边资源、教育主题、教学活动之间的结构化关联关系，支撑乡村学校开展在地化、生活化、实践化的思政教育。

### 4.2 分目标

1. 建立河北地区乡村学校基础数据库，精准采集学校名称、位置、所属区域等信息
2. 建立学校周边思政教育资源数据库，对资源进行分类、标注、审核和空间关联
3. 建立“学校-资源-主题-活动”组织机制，支持教师按主题生成教学素材和实践活动建议
4. 建立地图展示与检索能力，让教师能够从空间视角快速发现可用资源
5. 为后续知识图谱问答、智能推荐、教学辅助生成等功能预留数据基础

## 5. 建设范围

### 5.1 空间范围

- 省域范围：河北省
- 首期试点：可先选择 1 个地市或 1 至 2 个县区开展样例验证
- 后续扩展：逐步覆盖更多县、乡镇及乡村学校

### 5.2 对象范围

平台核心对象包括：

1. 学校
2. 思政教育资源
3. 学校与资源关系
4. 教育主题
5. 教学活动建议

### 5.3 资源范围

除已有红色文化资源外，平台后续重点纳入以下资源类型：

1. 红色文化资源
2. 非物质文化遗产资源
3. 地方传统文化资源
4. 乡贤文化资源
5. 爱国主义教育基地
6. 劳动教育资源
7. 志愿服务资源
8. 社会实践场所
9. 养老院、福利机构等公益服务场所
10. 村史馆、文化礼堂、农耕基地、生态教育点等其他可用于育人的在地资源

## 6. 目标用户

### 6.1 核心用户

1. 乡村学校教师
2. 学校管理者
3. 项目研究团队与数据采集团队

### 6.2 扩展用户

1. 教育行政部门
2. 志愿服务组织
3. 合作实践基地管理人员
4. 后续参与资源共建的社会机构

## 7. 核心业务理念

平台业务主线不再是“区域下有哪些红色资源”，而是：

**某所学校周边有哪些可服务思政教育的真实资源，这些资源能够围绕哪些主题进入课堂、形成哪些活动、产生哪些育人价值。**

因此平台的核心组织逻辑为：

`学校 -> 周边资源 -> 教育主题 -> 教学活动 -> 育人成果`

## 8. 功能需求

## 8.1 学校信息管理

### 8.1.1 功能目标

建立学校主数据体系，形成平台的一号业务对象。

### 8.1.2 需求内容

1. 录入学校基础信息，包括学校名称、所属地区、地址、学段、学校类型、是否乡村学校、是否教学点等
2. 采集学校经纬度坐标并支持地图定位
3. 支持坐标来源记录、人工校验与修正
4. 支持学校数据导入、批量维护和审核
5. 支持按地区、学段、学校类型进行检索与筛选

### 8.1.3 输出结果

形成标准化学校数据库，并能在地图上准确显示学校位置。

## 8.2 周边思政教育资源管理

### 8.2.1 功能目标

建立统一的本土思政教育资源库，将不同类型资源纳入同一数据框架。

### 8.2.2 需求内容

1. 录入资源名称、类别、简介、地址、坐标、图片、来源等基础信息
2. 支持资源大类与子类分类管理
3. 支持资源教育价值描述，例如爱国主义教育、传统文化传承、劳动教育、尊老敬老、社会责任等
4. 支持资源适用学段、开放情况、可预约情况、建议活动形式等信息记录
5. 支持资源审核、修改、停用和备注管理
6. 支持已有红色文化资源逐步映射为通用教育资源

### 8.2.3 输出结果

形成可检索、可分类、可视化展示的本土思政教育资源库。

## 8.3 学校与周边资源关联

### 8.3.1 功能目标

建立学校中心化的空间关联能力，回答“某所学校周边有哪些资源”。

### 8.3.2 需求内容

1. 支持按照学校周边一定范围自动检索资源
2. 支持记录学校与资源之间的距离、可达性、建议出行方式、预计活动时长等信息
3. 支持人工确认“可用于本校思政教育”的有效资源关系
4. 支持标注关系类型，如周边资源、合作资源、实践资源、课程资源
5. 支持按学校查看周边资源列表和主题聚合结果

### 8.3.3 输出结果

形成“学校-资源”结构化关联图谱，为地图推荐和教学组织提供基础。

## 8.4 地图展示与空间检索

### 8.4.1 功能目标

通过地图方式展示学校与资源的空间关系。

### 8.4.2 需求内容

1. 支持学校点位展示
2. 支持资源点位展示与分类图层切换
3. 支持点击学校查看周边资源分布
4. 支持点击资源查看详情、教育价值和活动建议
5. 支持按地区、资源类别、主题、学段进行地图筛选
6. 支持学校周边固定半径范围展示
7. 支持列表与地图联动

### 8.4.3 输出结果

形成可交互、可检索、可分析的学校周边资源数字地图。

## 8.5 教育主题组织

### 8.5.1 功能目标

将零散资源转化为可用于思政教育的主题化内容。

### 8.5.2 需求内容

1. 为资源打教育主题标签
2. 支持围绕学校自动或人工组织主题资源包
3. 支持按主题查看对应学校和资源
4. 支持形成典型主题示例，例如：
   - 学校周边的红色记忆
   - 村落里的非遗文化
   - 本地中华优秀传统文化课堂资源
   - 敬老志愿服务路线
   - 劳动教育实践点
5. 支持同一资源归属多个教育主题

### 8.5.3 输出结果

形成可直接服务课堂设计的主题化资源集合。

## 8.6 教学活动建议生成

### 8.6.1 功能目标

将资源从“可看”转化为“可用”。

### 8.6.2 需求内容

1. 针对学校与资源关系生成教学活动建议
2. 支持活动主题、适用年级、活动目标、活动流程、准备事项、安全提示、预期成果等内容组织
3. 支持课堂活动、校内活动、校外实践、志愿服务等不同活动类型
4. 支持基于单个资源或多个资源形成主题路线
5. 支持教师二次编辑和保存活动方案

### 8.6.3 输出结果

形成可供教师直接参考的活动方案和课堂组织建议。

## 8.7 检索、统计与分析

### 8.7.1 功能目标

支持资源发现、区域分析和项目管理。

### 8.7.2 需求内容

1. 支持按学校名称、地区、资源名称、资源类别、教育主题检索
2. 支持统计不同地区学校数量、资源数量、资源类别分布
3. 支持统计单校周边资源覆盖情况
4. 支持发现资源薄弱区域，为后续采集提供依据

## 8.8 数据采集与审核管理

### 8.8.1 功能目标

保证学校和资源数据真实、规范、可追溯。

### 8.8.2 需求内容

1. 支持 Excel 模板导入与后台人工录入
2. 支持记录数据来源、采集时间、采集人员
3. 支持草稿、待审核、已通过、已驳回等状态流转
4. 支持审核意见记录
5. 支持图片、链接、说明材料等佐证信息管理

## 8.9 后续扩展需求

以下功能可作为第二阶段或第三阶段扩展：

1. 学校周边资源智能推荐
2. 基于知识图谱的资源关系查询
3. 基于大模型的教学素材辅助生成
4. 问答式资源检索
5. 主题路线自动规划
6. 教师使用反馈与资源评价机制

## 10. 角色与权限需求

建议平台至少设置以下角色：

1. 系统管理员：维护基础数据、账号和全局配置
2. 数据采集人员：录入学校与资源信息
3. 审核人员：审核学校、资源和关系数据
4. 教师用户：查看地图、检索资源、使用活动建议
5. 研究团队成员：查看统计数据、导出分析结果

## 11. 非功能需求

## 11.1 数据质量要求

1. 学校坐标准确
2. 资源分类统一
3. 数据来源可追溯
4. 审核状态清晰
5. 信息更新可记录

## 11.2 性能要求

1. 地图页面可稳定展示学校和资源点位
2. 常用检索响应时间应满足演示和日常使用需求
3. 支持试点范围内批量导入和查询

## 11.3 可扩展性要求

1. 支持从少量样例学校扩展到更多地区
2. 支持从红色文化资源扩展到多类型本土资源
3. 支持后续接入知识图谱、问答服务和智能推荐

## 11.4 可维护性要求

1. 数据结构应与现有系统兼容，尽量采用增量扩展方式
2. 学校主轴与通用资源模型应尽量复用现有来源、媒体、标签和审核机制

## 12. 平台建设原则

1. 学校中心化：以学校为平台第一对象
2. 资源通用化：资源不再局限于红色文化
3. 教学导向化：所有资源组织都要服务课堂和育人
4. 小步迭代化：先试点、再扩展、再优化
5. 数据可追溯：所有重要内容应保留来源与审核信息

## 13. 建设阶段建议

## 13.1 第一阶段：试点闭环

目标：

先跑通“小范围学校样例 + 周边资源 + 地图展示 + 活动建议”的最小闭环。

建议任务：

1. 选取 3 至 5 所试点乡村学校
2. 完成学校坐标采集与校验
3. 为每所学校采集 5 至 15 个周边资源
4. 完成资源分类与审核
5. 建立学校与资源关系
6. 形成 3 至 5 个教学主题样例
7. 形成地图展示原型

## 13.2 第二阶段：区域扩展

目标：

将试点经验扩展到县域或地市范围。

建议任务：

1. 批量导入学校数据
2. 建立资源采集规范与审核流程
3. 完善主题组织和活动建议模板
4. 开展统计分析与区域比较

## 13.3 第三阶段：智能增强

目标：

在稳定数据基础上加入知识图谱和智能问答能力。

建议任务：

1. 扩展学校和通用资源的图谱建模
2. 增加智能检索与问答
3. 增加教学方案辅助生成
4. 增加路线推荐与主题聚合优化

## 14. MVP 最小可行版本建议

如果项目当前时间紧、任务重，建议先完成以下 MVP：

1. 学校信息管理
2. 通用资源信息管理
3. 学校与资源关系管理
4. 地图展示
5. 主题分类展示
6. 简单活动建议输出

暂时可以不优先做：

1. 复杂权限体系
2. 智能问答
3. 自动路线规划
4. 大规模知识图谱推理

## 15. 验收标准建议

项目阶段性验收可重点关注以下指标：

1. 是否形成学校主数据表和地图点位
2. 是否形成通用思政教育资源分类体系
3. 是否能按学校查看周边资源
4. 是否能按主题组织资源内容
5. 是否能输出至少一批可用的教学活动建议
6. 是否完成试点学校样例展示

## 16. 风险与难点

1. 学校位置数据可能存在缺失或坐标不准问题
2. 周边资源采集标准不统一，容易出现分类混乱
3. 部分社会实践资源开放性、合作性不稳定
4. 教学活动建议若仅停留在描述层，可能难以真正落地
5. 全省范围推进时，数据量与审核压力会明显上升

## 17. 下一步实施建议

建议按以下顺序推进：

1. 先确认平台一期 MVP 范围，避免一开始做得过大
2. 确定试点地区和试点学校名单
3. 统一学校采集模板和资源采集分类标准
4. 在现有数据库基础上新增 `school`、`local_edu_resource`、`school_resource_rel` 等核心表
5. 先录入一批真实样例数据，跑通地图展示和学校周边资源查询
6. 再补充主题组织和活动建议模块
7. 最后再考虑知识图谱扩展和智能问答增强

## 18. 结论

本项目第二阶段的关键，不是继续扩大单一红色文化资源库，而是完成平台逻辑从“资源中心”向“学校中心”的转变。

只有把学校作为核心坐标，把孩子身边真实存在的文化资源、社会资源和实践场景系统组织起来，平台才能真正服务河北乡村学校思政教育的现实需求，并形成具有推广价值的“空间化、在地化、生活化”思政教育新模式。
